## 1. 设计概述

### 1.1 设计目标

基于 CapsulaAI Phase 3 PRD，本设计文档为三大核心功能模块提供完整的界面设计方案：

1. **间隔重复系统 (Spaced Repetition)** - 帮助用户高效复习和巩固知识
2. **知识图谱自动维护 (Self-improving Graph)** - 自动发现、合并、优化知识图谱
3. **动态可执行洞察 (Actionable Insights)** - 主动提供基于个人知识的行动建议

### 1.2 设计原则

| 原则 | 说明 | 体现方式 |
|------|------|----------|
| **本地优先** | 数据隐私是核心 | 离线可用指示器、本地处理状态标识 |
| **简洁专业** | 知识管理工具属性 | 清晰的信息层级、减少视觉噪音 |
| **可用性优先** | 功能完整比美观重要 | 明确的操作反馈、一致的交互模式 |
| **渐进式复杂度** | 新手友好，专家高效 | 默认简化视图，高级功能可展开 |
| **上下文感知** | 智能但不打扰 | 洞察在合适时机出现，可控制频率 |

### 1.3 设计语言延续

延续 CapsulaAI 现有设计体系：
- **色彩**: 主色调保持专业蓝灰，功能模块使用辅助色区分
- **字体**: 系统字体栈，保证可读性
- **圆角**: 适度圆角 (8px-12px)，现代但不浮夸
- **阴影**: 细微阴影表示层级，避免过度装饰

## 8. 组件规范建议

### 8.1 通用组件

#### Card (卡片容器)
```
样式:
- 背景: white / dark:bg-gray-900
- 圆角: 12px (rounded-xl)
- 阴影: shadow-sm (hover: shadow-md)
- 边框: border border-gray-200 / dark:border-gray-800
- 内边距: p-6
- 过渡: transition-shadow duration-200
```

#### StatCard (统计卡片)
```
样式:
- 继承 Card 样式
- 数字: text-3xl font-bold
- 标签: text-sm text-gray-500
- 趋势: text-xs (绿色/红色指示)
```

#### PriorityBadge (优先级标签)
```
样式:
高优先级: bg-red-100 text-red-700 border-red-200
中优先级: bg-yellow-100 text-yellow-700 border-yellow-200
低优先级: bg-green-100 text-green-700 border-green-200
```

### 8.2 复习系统专用组件

#### ReviewCard (复习卡片)
```
样式:
- 尺寸: max-w-2xl mx-auto
- 最小高度: min-h-[400px]
- 正面/背面: 绝对定位，翻转动画
- 字体: 问题 text-xl, 答案 text-base
- 按钮: 评分按钮组，颜色渐变
```

#### Heatmap (学习热力图)
```
样式:
- 格子: w-3 h-3 rounded-sm
- 颜色等级:
  - level-0: bg-gray-100
  - level-1: bg-green-200
  - level-2: bg-green-300
  - level-3: bg-green-400
  - level-4: bg-green-500
- 悬停: 显示 tooltip 详细信息
```

### 8.3 维护系统专用组件

#### HealthScore (健康度分数)
```
样式:
- 进度条: 渐变色 (红→黄→绿)
- 分数: text-4xl font-bold
- 等级标签: 根据分数显示 优秀/良好/一般/需关注
```

#### MergePreview (合并预览)
```
样式:
- 双栏布局展示两个实体
- 箭头指示合并方向
- 影响列表: 图标 + 文字说明
```

### 8.4 洞察系统专用组件

#### InsightCard (洞察卡片)
```
样式:
- 左侧: 类型图标 + 颜色条
- 标题: font-semibold
- 描述: text-gray-600
- 操作栏: 按钮组 (主要/次要)
```

#### KnowledgeGapGraph (知识空白图)
```
样式:
- 进度条: 重要度可视化
- 状态图标: 已掌握/部分/未掌握
- 推荐路径: 步骤式流程图
```

## 9. 响应式设计考虑

### 9.1 断点设计

| 断点 | 宽度 | 布局调整 |
|------|------|----------|
| Mobile | < 640px | 单列布局，汉堡菜单，简化统计卡片 |
| Tablet | 640-1024px | 双列布局，侧边栏可折叠 |
| Desktop | > 1024px | 完整三列布局，固定侧边栏 |

### 9.2 移动端适配要点

**复习界面**:
- 卡片全屏显示
- 评分按钮增大触摸区域
- 手势支持: 左滑下一张，上滑翻转

**维护界面**:
- 任务列表改为卡片式
- 详情使用底部弹窗
- 合并预览改为上下布局

**洞察界面**:
- 洞察卡片垂直堆叠
- 操作按钮底部固定
- 详情使用全屏页面
